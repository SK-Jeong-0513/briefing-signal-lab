## 다음에 할 일 (우선순위)

1. **콘텐츠 전략 grill** — 핵심 구독자 페르소나 → 경제지에서 못 얻는 것 → 나만의 각도 → 필요한 1차 소스 순으로 "누구에게 무엇을 매일 준다" 약속 한 문장 도출.
2. **브리핑 시트 연동** — 약속 확정 후 브리핑용 구글 시트 구조(카테고리·날짜·제목·요약3줄·태그·출처) 설계 + 캘린더처럼 site.js/script.js가 시트 CSV 읽어 렌더.
3. **자동 러너 검토** — 별도 웹 불필요. GitHub Actions 크론/Apps Script로 1차 소스 수집→LLM 초안→시트 append(사람 에디터 판단이 핵심, 순수 AI 자동요약 지양).
4. 유료 폼 문항 추가(관심 키워드 + 추가 관심사) — 사용자 몫. 대시보드 실데이터, 커스텀 도메인은 선택.

## 결정/제약 메모

- Phase 1 = 결제·로그인·DB 없이 폼으로 수요 검증. 투자 조언 문구 금지.
- 배포는 `main` push → GitHub Actions 자동. `public/`만 서빙.
- 기존 IonQ/Telegram 자동화는 향후 "주식종목브리핑"용 — 현재 기술/금융/경제 브리핑엔 미사용.
- 캘린더 뷰토글에 `class="lang"` 쓰지 말 것(전역 KO/EN 리스너 하이재킹). `.cal-viewtoggle` 사용.

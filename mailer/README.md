# 주간 기술 브리핑 메일러 (Google Apps Script)

무료 구독자에게 주간 기술 브리핑(무료 부분)을 이메일로 발송하고, 이메일 링크로 선호도/수신거부를 응답 시트에 반영하는 샘플 메일러.

- **경로:** Google Apps Script (경로 2). GitHub Pages 사이트와 독립 — 사이트가 어디에 호스팅되든 무관.
- **발송:** `sendWeekly()` 수동 실행.
- **위치:** 이 폴더는 배포되지 않음(사이트는 `public/`만 서빙). 버전관리·복사용 원본.

## 설치 (약 10분)

1. **폼 응답 구글 시트** 열기 → **확장 프로그램 → Apps Script**.
2. `Code.gs` 전체를 붙여넣기.
3. 상단 **CFG** 확인/수정:
   - `SALT` → 아무 문자열로 교체(토큰 생성용, 노출돼도 큰 문제는 아니나 바꿀 것).
   - `COL` → 시트 헤더와 정확히 일치하는지 확인(현재: `이메일` / `관심 키워드` / `메일 수신 동의`). `관심 분야` 컬럼은 없으면 스크립트가 자동 생성.
   - `CONSENT_TRUE_INCLUDES` → 수신 동의 셀 값에 포함된 문자열(기본 `동의`). 폼 옵션 텍스트에 맞게.
4. **배포 → 새 배포 → 유형: 웹 앱** / 실행: 나 / 액세스: **모든 사용자** → 배포. 나온 **웹 앱 URL(/exec)**을 `CFG.WEBAPP_URL`에 붙여넣고 저장.
   - (웹앱을 안 만들면 수신거부는 mailto로 대체됨. 선호도 토글 링크는 웹앱 필요.)
5. 함수 선택 `sendWeekly` → 실행 → **최초 1회 권한 승인**.
   - `CFG.TEST_MODE = true`(기본)면 **나(OPERATOR_EMAIL)에게만 1통** 발송 → 미리보기.
6. 미리보기 OK → `CFG.TEST_MODE = false`로 바꾸고 다시 `sendWeekly` 실행 → **수신동의자 전원 발송**.

## 매주 발송

- `ISSUE` 객체만 그 주 내용으로 교체(무료 부분 = 사이트 `public/assets/content/tech.js`의 signals/headliner.summary와 동일).
- `sendWeekly` 실행. (수동 트리거 = 사람이 발행 스위치)

## 데이터 모델 메모

- **필터:** 수신 동의(`메일 수신 동의`에 `동의` 포함) O 인 구독자만 발송.
- **관심 키워드 vs 분야:** 폼은 *키워드*(HBM·CPO 등)를 받지만 브리핑은 *분야*(AI 인프라·반도체) 단위. 그래서:
  - `관심 분야` 컬럼이 비면 → 가동 분야(AI 인프라 + 반도체) **전체** 발송, 관심 키워드는 메일 상단에 표시.
  - 구독자가 메일의 분야 토글 링크를 누르면 → `관심 분야` 컬럼이 채워지고, 다음 발송부터 그 분야만.
  - 즉 키워드 → 분야로 자연 이행. (정식 오픈 시 폼 문항을 분야 선택으로 정리 권장)

## 개인정보/발송 규칙

- 이메일(개인정보)은 시트 밖으로 유출 금지. 링크에는 이메일 대신 **해시 토큰**만 사용.
- 수신거부 링크 필수(정보통신망법). `unsubscribe` 클릭 시 `메일 수신 동의` 셀을 `수신거부`로 변경 → 다음 발송에서 제외.
- 발송 한도: GmailApp 무료 ~100통/일. 지인 소수 샘플엔 충분. 대량·정식 단계는 도메인 인증 ESP(Resend/SES 등)로 이관.

## 정식 이관 시(향후)

커스텀 도메인 + 백엔드로 가면: SPF/DKIM/DMARC 건 전용 ESP 발송, 선호도·구독자는 실제 DB/API로. 이 Apps Script는 그때 대체되는 임시 샘플.
